## Whole-tugrik rounding note

All four NSO consumer-price-survey values pulled (5a–5d) came back as **whole-number tugrik** —
no fractional/möngö values appear anywhere in the raw API responses across every table queried in
this session (wages, exchange rates, SCC/NBFI financials, consumer prices). This is consistent
with the möngö (100th of a tugrik) being obsolete in everyday transactions, but I did **not**
independently confirm this via an authoritative statement (e.g. a Bank of Mongolia numismatics
page) — I could not reach a working non-SPA page on `mongolbank.mn` this session (see Blockers).
`[VERIFIED: the data-pattern observation itself, from raw API payloads; NOT VERIFIED: the
underlying legal/practical claim about möngö obsolescence, which rests on general knowledge, not
a primary source fetched this session]`. **Practical implication if re-deriving:** round all
re-derived MNT thresholds to whole tugrik; do not carry decimal-tugrik precision into the
requirements baseline.

## Decisions

- **D1 (unit inference on wage figures):** The NSO wage tables (`DT_NSO_0400_024V1.px`,
  `DT_NSO_0400_069V2.px`) return raw values like `"2726.9"` with no unit field in the JSON
  payload. I inferred **thousand MNT** because (a) NSO's standard convention for this series is
  мянган төгрөг (thousand tugrik), and (b) the alternative reading — raw MNT — would imply a
  national average monthly wage of MNT 2,726.9 (≈ USD 0.76/month), which is not plausible. I did
  not find an explicit unit label in the API response confirming this, so it is graded "medium"
  confidence on the unit even though the raw number itself is high-confidence. A follow-up task
  should confirm the unit against an NSO press release or the `1212.mn` UI (currently JS-broken)
  before this is load-bearing for a legal/regulatory threshold.
- **D2 (SCC/NBFI unit inference):** Same inference applied to the SCC and NBFI financial tables
  (Total Assets, paid-in capital, loan outstanding) — inferred as **million MNT** because the
  resulting Total Assets figure (MNT 389,961.7 million ≈ MNT 390 billion) matches the prior
  project research's independently-sourced "~MNT 390bn assets" almost exactly, which is strong
  circumstantial confirmation of the unit (not a stated unit in the API payload itself).
- **D3 (chose Q1 2026 as the SCC/NBFI reference period):** Used the most recent available quarter
  in each table rather than an older, possibly more widely-cited period, on the reasoning that a
  requirements baseline should anchor to current reality. The immediately-prior quarter's values
  are included in the raw pulls above (not tabulated) and are close in magnitude, so the choice of
  quarter is not high-stakes here.
- **D4 (did not pursue JS-shell/SPA obstacles further):** `mongolbank.mn`'s currency page and
  `frc.mn` both serve a client-rendered SPA shell for every route (confirmed by checking HTTP
  headers/content-type and diffing two different paths — both returned the same
  `text/html` shell). Per the task's method constraint, I stopped after confirming this rather
  than attempting to reverse-engineer their JS bundle's API base URL — that would cross from
  "primary source fetch" into circumvention. NSO's PxWeb API is not itself a proxy — its GET
  endpoints publicly document the table structure and its POST endpoint is the standard PxWeb v1
  query mechanism (documented, not a workaround) — so I used it directly.
