## Design decisions (what 04 implies vs inferred)

- **Staff console auth.** Every EP-12 operation and every EP-13 admin operation
  overrides the document default `memberOAuth2` with `security: - staffSSO: []`
  (root securityScheme; 04 ¬ß3.0 ‚Äî staff resources under `/api/v1/admin/...`).
  Member-facing EP-13 rows (capital-ledger, impact-scorecard, data-requests) and
  member EP-11 rows inherit the default. The EP-11 admin catalog CRUD
  (`/admin/notification-event-types`) uses `staffSSO`.
- **Maker-checker + audit.** 04 ¬ß3.12 states all EP-12 mutations are maker-checker
  + audit-logged. Each mutating op's summary/description notes this; where the
  contract needs the reference, responses surface an `approval_id`
  (MakerCheckerApproval, E-53), and `MakerCheckerApproval` + `MakerCheckerDecisionRequest`
  are modeled for `POST /admin/approvals/{id}/decision`.
- **Governance-entity collision guard.** Admin ballot/proposal/candidate schemas
  are PREFIXED `Admin...` (`AdminBallot`, `AdminBallotType`, `AdminBallotCreateRequest`,
  `AdminBallotCandidateRequest`, `AdminCandidate`, `AdminCandidatePatchRequest`,
  `AdminBallotCertification`, `AdminProposalQueueResponse`) so they never clash with
  EP-8's member-facing governance schemas at final assembly. Loan-domain enums are
  likewise prefixed (`AdminLoanDecision`, `AdminRestructureOrigin`, `AdminPledgeAction`).
- **ConfigurationParameter non-seed rule.** `ConfigurationParameter.value` and the
  create-request `value` are modeled as typed-but-schema-open JSON. NO seed values
  (share par, velocity/step-up limits) are baked as constants/defaults/enums ‚Äî
  they are runtime configuration. `key` examples are documented, not a closed enum.
- **Enums verbatim from 04.** All status/type enums copied verbatim from ¬ß2.10/¬ß2.11:
  NotificationCategory/Channel/DispatchStatus, StaffRole, StaffUserStatus,
  MakerCheckerStatus, ComplianceCaseType/Priority/Status, ReportRunStatus,
  DataSubjectRequestType/Status, AdminBallotType, and the decision-value enums.
- **Shared schemas $ref'd, never redefined.** `Money` (‚Üí `#/components/schemas/Money`),
  `Error` (via root `responses/*`), `MembershipStatus`, `KycStatus`, `DevicePlatform`
  are reused. `ComplianceCase` + its enums live in the EP-12 slice and are the target
  of the EP-13 AML alert queue (referenced conceptually, not duplicated).
- **Typing.** ids ‚Üí `format: uuid`; `*_at` ‚Üí `format: date-time`; money ‚Üí `$ref Money`;
  untyped/JSON-blob fields ‚Üí open objects (`additionalProperties: true`) marked INFERRED
  with a description. Every row's named error code(s) are attached to the operation;
  universal 401/403/429 per ¬ß3.0.
- **Project non-negotiables honored in prose.** Capital Ledger schema explicitly
  never describes deposits as insured/protected/guaranteed; QuietHours timezone note
  forbids hard-coded offsets (two zones, no DST); name fields use the
  three-field/DEC-35 display model, never first/last.

## Deliberate NON-writes (avoid duplicate-path / duplicate-schema at merge)

- `POST /api/v1/admin/members/{id}/status-transitions` ‚Äî the ¬ß3.12 row says

  `MemberStatusTransitionRequest/Response`). Not redefined here ‚Äî the assembly
  duplicate-path check would reject a second definition.
- "Dividend run administration | See ¬ß3.7 admin rows" ‚Äî those `/api/v1/admin/dividends/...`
  paths belong to EP-7 and are owned by that epic's slice.
