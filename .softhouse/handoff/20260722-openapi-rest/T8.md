## Design decisions (what 04 implies vs inferred)

- **Maker-checker + audit-log (04 §3.12).** Noted on every mutation. The run `approve` (a SECOND
  FINANCE_ADMIN, checker != maker) and `execute` steps ARE the maker-checker gate for a run —
  called out explicitly in both summaries. `approve` returns `DividendDeclaration` reflecting the
  MakerCheckerApproval resolution (04 says "via MakerCheckerApproval").
- **Money movement.** Only `execute` is a payout → carries `$ref '#/components/parameters/IdempotencyKey'`
  (04 §3.0). Its 409 covers both NOT_APPROVED and IDEMPOTENCY_CONFLICT. The other four dividend ops
  are money-neutral (create/calc/approve/reconcile) — no Idempotency-Key.
- **No formula / split / seed encoded.** The dividend calculation, the surplus split %, and the 10%
  community-grant allocation are ENGINE/config behavior. `community_grant_allocation`, `ratified_surplus`,
  `project_cap`, and all reconciliation/totals figures are `$ref Money` — amounts, never constants.
  `match_ratio_bps` is an integer basis-points input (04 names it `match_ratio_bps`); the `<= 10000`
  (1:1) rule is described in prose (drives 422 RATIO_ABOVE_1_TO_1), not baked as a default.
- **Typing.** `{id}` → `uuid`; `{fiscal_year}` → `integer` (E-45/E-32 fiscal_year is integer); `*_at` → `date-time`;
  money → `$ref Money`. Enums: `AdminCommunityProjectReviewDecision {APPROVE, DECLINE}` verbatim from 04;
  status uses the EP-7 `DividendDeclarationStatus` / EP-9 `CommunityProjectStatus` enums (reused, not
  redefined). Untyped 04 keys (`exceptions[]`, `deltas[]`, `reasons`) → typed guess + INFERRED note.
- **INFERRED status codes** (04 pins behavior, not HTTP status): 201 on the two creates; 202 on
  `execute` (04 names async EXECUTING + live progress, not the status); 200 elsewhere. Marked in-file.
- **404 on run/pool `{id}`/`{fiscal_year}` ops** added as a standard resource-lookup error even where the
  04 row lists no error (common codes apply everywhere, 04 §3.0). The `review` op (04 lists no specific
  code) carries 401/403/404 + 422 (body validation) only.
