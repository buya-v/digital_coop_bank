## Decisions

- **DeferredEnum: none needed.** Every status/type/enum in E-2,3,4,9-13 has its
  value set fully enumerated in 04 §2, so all are real `enum.Enum`. No value was
  invented; no field fell back to DeferredEnum.
- **Timestamps mixin vs append-only.** Applied the `Timestamps` mixin (created_at/
  updated_at) to KycSubmission, DeviceBinding, SavingsGoal, GroupPot,
  GroupPotMember, GroupPotApprovalRequest — matching the E-5 MembershipShare
  precedent (mixin + domain `*_at` columns). ConsentRecord (E-4, "append-only")
  and GroupPotApprovalDecision (E-13, an immutable decision event) use
  `UUIDPrimaryKey` only + their own event timestamp, mirroring the LedgerEntry
  append-only precedent in ledger.py.
- **Nullability = mechanical.** A column is Optional iff 04 marks it `?`; every
  other field is NOT NULL. 04 does not otherwise state nullability.
- **JSON fields** (`ocr_extracted_fields`, `result_reasons`, `evidence_refs`,
  `auto_transfer`, `recipient`, `auth_context`) use SQLAlchemy `JSON`. Encryption
  noted by comment only — it is a service concern, not modelled.
- **`persona_inquiry_id` kept verbatim** from 04 §2 despite CLAUDE.md flagging it
  as a US-market vendor leak (Persona). In scope I mirror 04; a comment records
  the concern and that the Mongolia migration will revisit the eKYC vendor.
- **String lengths** from 04 where given (name 60/80, purpose 280); other String
  lengths chosen reasonably (see Unverified).

## Unverified (inferred types / choices)

- `updated_at` on the six Timestamps-mixin tables is mixin-provided; 04 lists only
  `created_at` (SavingsGoal, GroupPot) or domain `*_at` (others), never
  `updated_at`. Inferred, per the mixin/proof-model pattern. Noted in code.
- String lengths not given by 04: `persona_inquiry_id` 128, `device_fingerprint`
  255, `push_token` 255, `emoji_or_image_ref` 255, ConsentRecord `version` 64,
  `channel` 64. Reasonable defaults; noted inline.
- `submitted_at`/`bound_at`/`last_seen_at`/`invited_at`/`expires_at`/`decided_at`/
  `recorded_at` typed `datetime.datetime` (04 says "Timestamp"); `target_date`
  typed `datetime.date` (04 says "Date").
- No DeferredEnum used (see Decisions) — nothing deferred in this slice.
