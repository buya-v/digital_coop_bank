# T1 — DB engine + session factory + FastAPI session dependency

Branch: `softhouse/T1-session-plumbing`
Author role: Softhouse persistence-layer author (T1)
Scope: SYNC SQLAlchemy 2.0 + psycopg3 plumbing only. No feature model, no
route, no `main.py`/`config.py` change.

## Changes Made

- **New `backend/app/db/session.py`.**
  - `DatabaseNotConfigured(RuntimeError)` — raised when DB access is attempted
    with an empty `DATABASE_URL`; message points at `docker-compose.yml`'s
    `postgresql+psycopg://...` convention rather than surfacing a raw
    SQLAlchemy URL-parse error.
  - `_engine: Engine | None` / `_sessionmaker: sessionmaker[Session] | None` —
    module-level memoisation cells, both `None` at import time. Nothing at
    module scope calls `create_engine` or reads `DATABASE_URL`.
  - `engine_configured() -> bool` — `bool(get_settings().database_url)`. Pure;
    never connects.
  - `get_engine() -> Engine` — builds (once) via `create_engine(get_settings().
    database_url)` on first call; raises `DatabaseNotConfigured` first if the
    URL is empty, so the failure mode is a clear typed exception, not a
    downstream connection error.
  - `get_sessionmaker() -> sessionmaker[Session]` — builds (once) a
    `sessionmaker(bind=get_engine(), class_=Session, expire_on_commit=False)`.
- **New `backend/app/api/__init__.py`** — one-line package-marker docstring,
  no content (matches `backend/README.md`'s "no feature routers yet").
- **New `backend/app/api/deps.py`.**
  - `get_session() -> Iterator[Session]` — a FastAPI generator dependency.
    Raises `DatabaseNotConfigured` (uncaught, propagates to the caller) if
    `not engine_configured()`; otherwise instantiates a session from
    `get_sessionmaker()()`, `yield`s it, and `close()`s it in a `finally`. Does
    **not** translate the exception to an HTTP status — per the task spec,
    that mapping is the calling route's job (T2), since different routes may
    want different status codes (503 vs 500) for the same condition.

No other file touched. `main.py`, `config.py`, and every file under
`app/models/` are untouched — confirmed by `git diff --stat` below.

## Decisions

- **Async is explicitly deferred, not adopted.** `session.py`'s module
  docstring states this directly: `create_async_engine` / `AsyncSession` are a
  real architectural choice the scaffold has not made, and introducing either
  here would silently commit the whole codebase to it. Nothing in either new
  file imports `asyncio`, uses `async def`, `await`, `create_async_engine`, or
  `AsyncSession` — grepped, see Tests.
- **`SessionLocal` as a bound name vs. `get_sessionmaker()`.** The task asked
  for "a `SessionLocal` sessionmaker bound lazily," but a module-level
  `SessionLocal = sessionmaker(bind=get_engine(), ...)` would have to call
  `get_engine()` at import time — which would raise on this machine (no
  `DATABASE_URL`) and violate "importing this module must never connect and
  never raise." I did not create a literal `SessionLocal` name. `
  get_sessionmaker()` is the memoised equivalent the task's own text
  anticipated ("expose `get_sessionmaker()`... bound to `get_engine()`") and
  is the only session-factory surface. Flagging this explicitly in case the
  reviewer wants a `SessionLocal = get_sessionmaker` alias added — I left it
  out to keep the module's public surface to exactly the four names the task
  enumerated (`DatabaseNotConfigured`, `engine_configured`, `get_engine`,
  `get_sessionmaker`).
- **URL is passed through verbatim, no scheme rewriting.** `get_engine()`
  calls `create_engine(get_settings().database_url)` with no mutation.
  [VERIFIED: read `backend/docker-compose.yml`] the existing convention
  already sets `DATABASE_URL=postgresql+psycopg://dcb:dcb@db:5432/dcb` — the
  psycopg3 dialect prefix is the operator's/compose config's responsibility,
  not something this module should silently rewrite from a bare
  `postgresql://`.
- **`DatabaseNotConfigured` lives in `session.py`**, not a separate errors
  module — it's one class, used by exactly two files in this slice; a
  dedicated module would be one class in an otherwise-empty file.
- **No HTTP-status mapping in `deps.py`** — per the task's explicit
  instruction, left for T2's route layer.

## Follow-ups

- T2 (or whichever task adds the first real route) needs to decide the
  `DatabaseNotConfigured → HTTP status` mapping (503 Service Unavailable
  seems the obvious choice, but that's a route-layer decision, not this
  task's).
- When `main.py`'s `/ready` endpoint grows a real DB check (its own docstring
  already flags this as pending), it should use `engine_configured()` +
  `get_engine()` from this module rather than duplicating engine
  construction.
- If a reviewer wants a literal `SessionLocal` symbol for naming-convention
  familiarity, it can be added as `SessionLocal = get_sessionmaker` (a
  same-signature alias) without changing behaviour — flagged above under
  Decisions, deliberately left out of this pass to keep the changeset to
  exactly the four names requested.
- Async remains deferred, not ruled out. If/when the project adopts an async
  stack (e.g. to match an async web framework upgrade), this module is the
  one place that needs a parallel `async_session.py`; nothing here should be
  retrofitted with `async def` methods.
